Dos hobbits están tratando de atrapar una gallina en el patio. Esto debería ser una tarea fácil, pero estos dos vecinos nunca se han llevado bien, y aunque la gallina anda suelta y necesitan atraparla, ellos se rehúsan a hablarse entre sí. Por esta razón, tendremos que encontrar un algoritmo que funcione para los dos hobbits de forma independiente, de manera que cualquiera de ellos para atrapar la gallina sin tropezarse con una cerca, con un árbol o entre sí. Una última cosa, los hobbits no puede correr o esperar por siempre. Después de todo, atrapar una gallina es importante, pero ante todo deben adherirse a su estricto horario de comidas.

Se te dará un mapa del jardín como una secuencia de cadenas (string)donde::
- "." es una celda vacía;
- "I" es uno de los Hobbit;
- "S" es el compañero;
- "C" es la gallina (nuestro objetivo);
- "X" es un obstáculo (un árbol, una roca, etc.)

El patio está rodeado por una cerca.

La gallina y los Hobbits pueden moverse en 8 direcciones hacia las celdas vecinas (incluyendo diagonales), o pueden esperar y permanecer en el mismo lugar por un tiempo. Las acciones que pueden ejecutarse desde cada celda se describen a continuación:
- "N" norte, "S" sur, "W" oeste, "E" este;
- "NW" noroeste, "NE" noreste, "SW" suroeste, "SE" sureste;
- "" (empty string) es esperar.

La gallina es impredecible y puede utilizar diversas tácticas (incluyendo el movimiento aleatorio), pero no puede salir del patio. Los hobbits no pueden atravesar obstáculos o cercas, y no deben encontrarse entre sí, incluso justo antes de atrapar a la gallina.

Ahora viene la parte difícil. Tu función deberá ser utilizada dos veces en cada paso - una vez para el primer Hobbit y una vez para el segundo. Los Hobbits no pueden comunicarse entre sí y hacen sus movimientos de forma simultánea, así que debes tener cuidado de no mover los dos Hobbits hacia una misma casilla.

Para ganar puedes trasladar cualquier Hobbit a la celda en la que se encuentra la gallina, pero recuerda que sólo uno debe hacer el movimiento final, ya que si entran a la misma celda simultáneamente, se considera como un choque (y no está permitido). Los hobbits y la gallina se mueven al mismo tiempo, así que ten en cuenta que si acabas de llegar a la celda donde estaba la gallina, no hay garantía que puedas atraparla. Dado que la hora de comer se acerca, solo tienes 100 movimientos para atraparla.

Esta es una misión "multi-llamada" y tu función será invocada hasta que ganes o pierdas. Cada ciclo de pasos se ejecuta en un nuevo entorno, por lo que puede utilizar variables globales entre cada paso. Sin embargo, una vez se empieza una nueva “cacería” todas las variables serán reseteadas.

Este mapa se verá un poco distinto después de cada llamada a la función:

"......",   "......",
".I.XX.",   ".S.XX.",
"...CX.",   "...CX.",
".XX.X.",   ".XX.X.",
"...S..",   "...I..",
"......",   "......",

Datos de Entrada: El mapa del patio como una tupla de cadenas (string). Tu función será llamada dos veces por cada paso.

Salida: La acción a seguir como una cadena (string). La acción debe ser una de las siguientes: "N", "S", "W", "E", "NW", "NE", "SW", "SE", "S".

Ejemplo: 

hunt(("......",
      ".I.XX.",
      "...CX.",
      ".XX.X.",
      "...S..",
      "......")) # return an action

¿Cómo se usa?: Este concepto es utilizado en robots automatizados que trabajan en equipo sin que implique una conexión directa entre ellos, como robots de fabricación o Cylons. Dichos robots deben implementar una AI lo suficientemente compleja que les permita "pensar" y predecir las acciones de otros objetos.
